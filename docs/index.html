Ihre Miethäuser,
geführt wie ein Kontobuch.

Ich habe Vermio gebaut, weil brauchbare Hausverwaltung entweder ein Abo kostet oder Mieterdaten in fremde Clouds lädt. Es läuft auf dem eigenen Rechner und rechnet Nebenkosten auf den Tag genau — nicht geschätzt.

Blatt 01

Was Vermio führt

Nebenkosten

Tagesgenau abrechnen

Strom, Gas, Wasser, Heizung, Betriebskosten. Jede Kostenart wird auf die effektive Nutzungszeit umgelegt und als fertiges A4-PDF ausgegeben.

Mieten

Im Blick behalten

Wer hat gezahlt, wer nicht — pro Monat und Objekt. Rückstände erkennt Vermio selbst, die Jahresbilanz rechnet Rendite je Wohnung.

Mahnungen

Sauber schreiben

Zahlungsaufforderung als PDF — mit korrekter Anrede, allen Mitmietern im Adressfeld und eingebetteter Unterschrift.

Blatt 02

Umlegen, nicht schätzen

In Excel entstehen die Fehler beim Einzug mitten im Jahr oder beim Mieterwechsel im Abrechnungszeitraum. Vermio nimmt die Vertragsdaten und rechnet die Nutzungszeit exakt auf den Tag — für jede Kostenart mit ihrem eigenen Zeitraum.

  • 01Eigener Abrechnungszeitraum pro Kostenart (Anbieter ≠ Mietjahr)
  • 02Effektiver Mietzeitraum automatisch aus den Vertragsdaten
  • 03Zählernummern direkt aus der Zählerverwaltung übernommen
  • 04Warmwasser separat: Frischwasser + Abwasser + Heizenergie
  • 05Heizkostenverteiler (ISTA): Einheiten → kWh umgerechnet
  • 06Abrechnungsprofile: Werte einmal eingeben, nächstes Jahr laden
  • 07WG-Support: geteilte Zähler, anteilige Kosten, Personenzahl automatisch

Blatt 04

Einrichten

Braucht nur Docker Desktop. Drei Schritte, dann laufen Datenbank, API und UI im Container.

Terminal
# 1 · Holen
git clone https://github.com/slgao/landlord_system.git
cd landlord_system

# 2 · Konfigurieren
cp .env.example .env      # POSTGRES_PASSWORD setzen

# 3 · Starten — baut und startet alle Container
make up
Next.js UI → localhost:3000 FastAPI → localhost:8000 · docs /docs Schema legt Alembic beim Start selbst an

Auf mehreren Rechnern

Einmal zu Neon migrieren (das kostenlose Tier reicht), dann läuft dieselbe Datenbank auf Mac und Linux gleichzeitig.

Terminal
# Einmalig: lokalen Docker zu Neon migrieren
./scripts/migrate_to_neon.sh "postgresql://...neon..."

# Lokale Kopie vom Stand der Neon-DB aktualisieren
./scripts/sync_local_from_neon.sh
